The flowchart represents the operational workflow of the elastic resource allocation mechanism within the proposed system. Initially, incoming traffic is classified based on priority and assigned to appropriate queues. This classification step ensures that critical data is identified early in the processing pipeline. Following classification, the system evaluates available edge resources and dynamically allocates computational capacity to handle the incoming workload.

When local edge resources are insufficient to handle the traffic load, the system triggers a cloud offloading mechanism. This ensures that excess workload is redirected to cloud servers, preventing congestion and maintaining service continuity. The decision-making process is adaptive, allowing the system to respond to real-time variations in traffic intensity and node density.

The workflow highlights the importance of elasticity in modern network systems. By continuously monitoring resource utilization and adjusting allocations accordingly, the system achieves efficient load balancing. This approach not only improves performance but also enhances reliability, making it suitable for large-scale IoT environments.
